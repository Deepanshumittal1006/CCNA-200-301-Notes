# CCNA 200-301 Lab Notes

# Day 3 Lab – Examining Network Traffic with Packet Tracer Simulation Mode

## Lab Objective

The purpose of this lab is to use Cisco Packet Tracer's **Simulation Mode** to observe network traffic and understand how the **OSI Model** and **TCP/IP Model** operate in real network communications.

By the end of this lab, you will be able to:

* Use Packet Tracer Simulation Mode
* View protocol traffic moving through a network
* Observe Layer 1 through Layer 7 information
* Understand encapsulation in real network traffic
* Identify Layer 1, Layer 2, Layer 3, Layer 4, and Layer 7 protocols
* Generate DHCP traffic for packet analysis

# Examining STP Traffic

## Protocol

```text
STP
(Spanning Tree Protocol)
```

---

## Purpose

Prevents Layer 2 switching loops in Ethernet networks.

---

## Observation

A packet generated by SW2 shows:

### Layer Information Present

| Layer   | Present |
| ------- | ------- |
| Layer 1 | Yes     |
| Layer 2 | Yes     |
| Layer 3 | No      |
| Layer 4 | No      |
| Layer 5 | No      |
| Layer 6 | No      |
| Layer 7 | No      |

---

## Layer 2 Information

Packet Tracer displays:

```text
IEEE 802.3 Header
```

This confirms:

```text
Ethernet = Layer 2 Protocol
```

---

## Encapsulation Example

Simulation Mode displays:

```text

```

This demonstrates the encapsulation process discussed in the TCP/IP lesson.

## Layer 1 Information

# DHCP Packet Analysis

The DHCP packet contains information from multiple layers.

---

## Layer Information Present

| Layer   | Present |
| ------- | ------- |
| Layer 1 | Yes     |
| Layer 2 | Yes     |
| Layer 3 | Yes     |
| Layer 4 | Yes     |
| Layer 7 | Yes     |

---

## Why Are Layers 5 and 6 Missing?

The TCP/IP model combines:

```text
Layer 5
Session

Layer 6
Presentation

Layer 7
Application
```

into a single:

```text
Application Layer
```

Therefore Packet Tracer typically shows only Layer 7 information.

---

# Layer 4 Analysis

DHCP uses:

```text
UDP
(User Datagram Protocol)
```

Packet Tracer displays:

```text
The device encapsulates the PDU into a UDP segment.
```

Important CCNA Note:

UDP operates at:

```text
Layer 4
```

---

# Encapsulation Review

When DHCP data is transmitted:

### Layer 7

Creates DHCP message

↓

### Layer 4

UDP header added

↓

### Layer 3

IP header added

↓

### Layer 2

Ethernet header and trailer added

↓

### Layer 1

Bits transmitted

| ------------------- | ------- |
| DHCP                | Layer 7 |

---

# Key Concepts Demonstrated

## Encapsulation

Data moves down the stack and headers are added.

---

## Decapsulation

Receiving devices remove headers as data moves up the stack.

---

## Layered Communication

Different protocols operate at different layers.

Examples:

* STP → Layer 2
* OSPF → Layer 3
* DHCP → Layer 7
* UDP → Layer 4
